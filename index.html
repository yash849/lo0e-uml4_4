UML4.4 Git & Use case diagram

Introductie

Doelstelling

  • Behendig zijn in: commits, branching, merging, pushing en pulling
  • Use case diagrammen kunnen maken

Startinstructies

  • Fork deze repository naar jouw eigen github account
  • Clone deze fork naar jouw computer
  • Open je favoriete git applicatie en editor.
  • Volg de instructies!!!!!

Afbeeldingen toevoegen

Opdracht

Voor je begint met het aanpassen van de code, zorg dat je in de develop branch zit.

  • Maak een nieuwe branch: afbeeldingen
  • Controleer dat je in deze branch zit
  • Vervang bovenaan de pagina "Naam van student" door jouw eigen naam (id=naam)
  • Commit deze wijziging
  • Zoek een passende afbeelding voor DIT hoofdstuk, voeg deze afbeelding toe in de map images en maak een commit. zoeken
  • Pas de src aan van de placeholder afbeelding van DIT hoofdstuk aan, zodat de nieuw toegevoegde afbeelding zichtbaar is en maak een commit.
  • Ga terug naar de develop branch.
  • Merge de aanpassingen van afbeeldigen naar develop.
  • Push de develop branch naar github

Use case diagram

  • Ga naar de develop branch.
  • Start een nieuwe feature branch: use-case-diagram
  • Maak een Use Case Diagram voor de onderstaande User-stories
    Maak hierbij gebruik van:
    • Include relaties
    • Extend relaties
    • Primaire & secundaire actor(s)
  • Vervang onderstaande placholder image (id=use-case-image) met jouw use case diagram.
  • Commit deze aanpassing
  • Merge de aanpassingen van use-case-diagram naar de develop branch
  • Push de develop branch naar github

User stories

  • Als koper wil ik een account aanmaken zodat ik mijn bestellingen kan bijhouden
  • Als koper wil ik inloggen zodat ik producten direct kan bestellen
  • Als koper wil ik producten zoeken zodat ik mijn favoriete product kan bestellen
  • Als koper wil ik mijn bestellingen betalen met ideal omdat ik geen creditcard heb.
  • Als koper wil ik mijn bestellingen betalen met creditcard zodat mijn aankopen beschermd zijn.
  • Als koper wil ik mijn vragen kunnen stellen aan de service-medewerker zodat ik mijn vragen beantwoord krijg.
  • Als beheerder wil ik producten toevoegen zodat kopers nieuwe producten kunnen bestellen.
  • Als beheerder wil ik producten verwijderen zodat oude producten niet meer te bestellen zijn.
  • Als beheerder wil ik producten wijzigen zodat de product informatie up-to-date blijft.
